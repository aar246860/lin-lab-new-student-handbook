# 第一章 規則與辦法（須知）

本章保留舊手冊的章節順序與「須知」格式。除私人資訊、舊路徑、聯絡資訊與不宜公開內容外，文字只做必要修正；舊工具名稱原則上保留，並在必要處以括號標示目前可用的替代工具。

## 1.1 老師給的備忘

### 1.1.1 每週師生會面要點

1. 會面前
   - 將本週進度寫入固定進度檔。舊版以 doc 檔夾檔寄給老師；本版可用 `weekly-note.md`、共用資料夾或 GitHub issue。
   - 將研究內容整理、分析與圖表一起準備。不要只帶口頭說明。
   - 若碰到困難，須先寫下問題與可能解決途徑，並提出可行方法。
   - 若本週主要工作是閱讀文獻，須列出文獻題名、作者、年份、研究問題、使用方法、與本研究的關聯。

2. 會面時
   - 會面資料要帶齊，包括文獻、計算手稿、程式輸出、圖表、草稿或其他可討論資料。
   - 討論時以資料、圖與文字為主，不以印象或記憶為主。
   - 每次會面結束前，須確認下週要完成的工作。

3. 會面異動
   - 若排定時間無法會面，須事先通知並另約時間。
   - 期中、期末或長假期間可視情況調整，但仍須留下進度紀錄。
   - 若週間離開研究室兩天以上，須事先告知老師目前工作狀態與返校時間。

### 1.1.2 借用圖書、期刊及資料

1. 借用研究室紙本資料、書籍、期刊、手稿或掃描檔，須留下借用紀錄。
2. 借閱資料不得任意塗寫。紙本資料避免使用會留下痕跡的筆。
3. 借用資料原則上於一週內歸還；如需延長，須更新紀錄。
4. 看完資料須歸回原位。若資料已數位化，須把檔名、來源與位置記清楚。
5. 大量影印、掃描或轉存資料前，須先確認是否合乎授權與研究室管理規則。

## 1.2 新生暑期工作

新生暑期工作以建立研究室基本工作能力為主。舊手冊要求新生先讀手冊、學工具、蒐集資料，再開始讀書報告；本版保留此順序。

1. 閱讀資料
   - 研究室手冊。
   - 論文寫作、研究報告格式、科學論文寫作、科技英文、時間管理與會議簡報相關資料。
   - 研究方向相關論文與老師指定資料。

2. 個人工作環境
   - 確認座位、電腦、帳號、網路、共用資料夾與備份位置。
   - 建立個人研究資料夾，並依研究室命名規則整理。

3. 蒐集研究方向資料
   - 依研究題目蒐集期刊論文、研討會論文、學位論文、技術報告與資料集。
   - 每篇文獻至少留下題名、作者、年份、問題、方法、主要結果與備註。

4. 學習研究工具
   - 電腦硬體與作業系統基本維護。
   - 文書處理：Word、PowerPoint、Excel、PDF 工具。
   - 繪圖工具：Grapher、Surfer、Tecplot、Visio、MATLAB 或 Python 繪圖套件。
   - 程式工具：Fortran、MATLAB、Mathematica、Python、Jupyter。
   - 文獻工具：EndNote、Google Scholar、Web of Science、Zotero 或 BibTeX。
   - 資料與網路工具：FTP、雲端硬碟、GitHub、壓縮工具、掃描與 OCR 工具。
   - 地下水分析工具：抽水試驗分析軟體、數值模式、統計或可靠度分析工具。

5. 讀書報告
   - 十月起開始讀書報告。
   - 報告不是摘要翻譯，須說明論文的問題、方法、假設、主要結果、限制與可借用之處。

## 1.3 暑假、春節與學期開學前準備工作

1. 開學前與老師討論修課、旁聽課、研究進度與本學期目標。
2. 回顧前一學期或半年研究工作，整理已完成項目、未完成問題與下階段工作。
3. 提供文獻回顧清單、閱讀心得、研究成果、展望與修正後進度表。
4. 發展中的程式須撰寫文件，至少包括：
   - 程式目的。
   - input 檔與 output 檔內容。
   - 主要變數定義。
   - 執行方式。
   - 已知限制。
5. 整理個人硬碟、雲端資料夾、文獻資料庫、程式與圖表。
6. 假期前須備份研究程式與數據檔，並交代回來日期與期間可否聯絡。

## 1.4 研究生、助理離校前須知

離校、畢業、轉題或離開計畫前，須完成交接。交接資料至少包括：

1. 借用圖書、期刊、資料與研究室物品歸還紀錄。
2. 論文與計畫相關最新版檔案。
3. 參考文獻清單與 PDF 或掃描檔位置。
4. 圖、表與製圖資料檔。
5. seminar、研討會、口試與期刊投稿簡報。
6. 解析解、推導、計算手稿或 notebook。
7. 程式 source code、input、output、readme 與執行環境說明。
8. 所有重要初稿、改稿與定稿版本。
9. 未完成問題與不應繼續使用的錯誤版本。
10. 若屬計畫成果，須完成成果報告、精簡報告或老師指定之交接文件。

## 1.5 檔案與程式管理須知

### 1.5.1 程式

1. 程式變數力求統一。主程式前應定義重要變數。
2. 類似功能之程式應分類存放，以便日後或他人參考。
3. input 資料格式應盡量統一；必要時另寫格式轉換程式。
4. output 檔應 echo print input，亦即在輸出檔內列出本次使用之輸入檔名、主要參數與條件。
5. 盡量避免 interactive mode，以減少輸入錯誤與日後重跑困難。
6. 程式 readme：程式建立後須寫 readme。readme 至少包括：
   - 程式目的。
   - source code 檔名。
   - input 檔名與格式。
   - output 檔名與格式。
   - 參數說明與數值範圍。
   - 變數定義。
   - 參考文獻或推導來源。

### 1.5.2 檔案命名與存放

可用四層命名原則檢查檔名：日期、主題、動作、版本。若是文獻檔，則至少要能看出作者、關鍵字、年份與期刊。

1. 期刊論文或文獻檔名須能看出作者、題目關鍵字、年份與期刊。
2. 研討會文件可用第一作者、關鍵字、年份、會議簡稱、摘要或全文標示。
3. 關鍵字以二至三個容易判定內容的字為原則。
4. 電子文獻依期刊、年份或主題分類，不任意散放。
5. 給老師或共同作者修改之文稿，檔名後依修改順序加版本碼，例如 a、b、c，定稿後拿掉暫時字尾。
6. 若為審稿後修改稿，可加 R1、R2 等版本碼。
7. 不使用 `final_final_v3_new` 這種無法判斷來源與用途的檔名。

### 1.5.3 Data

1. 原始資料須保留，不得直接覆蓋。
2. 重要資料須有兩份以上備份，一份由個人保存，一份放在雲端、共用硬碟或 GitHub。
3. 若資料經清理、插值、篩選或轉換，須保留處理程式與處理紀錄。
4. 資料檔旁須有簡短說明，註明來源、單位、欄位、時間、座標與限制。

## 1.8 研究生的權利與義務

1. 研究室提供基本研究資源，包括手冊、共用資料、軟硬體與必要工作環境。
2. 研究生與助理須共同維護研究室空間、書籍、期刊、電腦、儀器、共用資料夾與 seminar 安排。
3. 公用資料不可只存在個人電腦。若資料與計畫、論文或學生訓練有關，須放在可交接位置。
4. 每週須留下研究進度紀錄。

## 1.9 Group Seminar 主持人、報告人及安排人須知

1. 主持人
   - 有事不能主持時，須提前找代理人。
   - 報告前確認投影、電腦與時間。
   - 控制報告時間，必要時提醒報告人。
   - 記錄主要問題、建議與待修正事項。

2. 報告人
   - 報告別人的論文時，須說明 real-world problem、背景、作者使用的方法或理論、重點、優缺點、貢獻與創新。
   - 讀完文獻後須思考：方法能否改進？優點能否吸收？限制是否與自己的研究有關？
   - 報告前須準備投影片，投影片原則為 keywords、字體要大、每張不超過八行。
   - 重要或高度相關文獻須整理並放入文獻資料庫。

3. 安排人
   - 事先排定報告順序與主持人。
   - 確認場地、設備與報告時間。
   - 若報告時間需更改，須提前告知。

## 1.10 Presentation 須知

1. 報告前先考慮內容、大綱、研究主題、重點、貢獻與未來方向。
2. 新手可先寫較詳細原稿；有經驗後可改成大綱式提示。
3. 投影片原則：
   - 一張投影片一個重點。
   - 多用圖，少用長句。
   - 圖表不要太複雜。
   - 字體要大，對比要明顯。
   - 每張投影片最好不超過八行。
   - 每張投影片要有標題。
   - 不使用過度花俏動畫。
4. 報告前須預演，檢查播放設備、燈光、檔案路徑與備份檔。
5. 報告時注意時間、音量、速度、站姿與雷射筆使用方式。
6. 問答時若不能立即回答，須先確認問題，記錄後再回覆。

## 1.11 論文的學術倫理

學術研究最重要的是真實。本研究室要求：

1. 數據不可修改、捏造或選擇性呈現。
2. 外投稿件不可一稿兩投，也不可把沒有新意的相同內容重複投稿。
3. 署名須依實際貢獻，不因職位或關係任意署名。
4. 相關文獻須明確引用與討論，尤其是與本文高度相關的前人工作。
5. 使用他人圖、表、資料或文字，須取得必要授權並清楚引用。
6. AI Agent 可協助整理與改稿，但不可取代資料查核、文獻引用與作者責任。
