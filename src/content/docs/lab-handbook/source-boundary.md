# 來源邊界

這份新版手冊的參考來源是一批舊版地下水研究室手冊照片。照片內容包含目錄、研究室規則、寫作建議、seminar 規範、工具操作、研究方法與雜項說明。它的價值在於提供一份「研究室如何訓練新人」的詳細骨架，但它不是林穎凡研究室的現行制度，也不能原封不動公開。

## 三層資料

| 層級 | 位置 | 是否公開 | 用途 |
|---|---|---|---|
| 原始照片 | 本機桌面的 `GW guide` 資料夾 | 否 | 作為歷史手冊來源，不放入 repo |
| OCR 草稿 | `.gwguide-private/ocr` | 否 | 幫助重建章節與內容，不直接當正式文字 |
| 改寫後手冊 | `src/content/docs/lab-handbook/` | 可公開 | 給學生閱讀，已改成目前研究室工作方式 |

## 為什麼不能直接貼 OCR

OCR 會有錯字，也可能把頁面背面透出的字、舊式路徑、通訊錄、舊研究室制度、人名與過時工具一起抓進來。這些內容如果直接公開，會有三個問題：

1. 讀者會誤以為它是目前研究室正式規定。
2. 舊手冊中的私人或內部資訊可能被不必要地曝光。
3. OCR 錯字會讓學生學到錯誤名詞或不自然的中文。

因此流程是：先 OCR、再人工判讀、再改寫成目前研究室語境，最後才放到公開手冊。

## 舊手冊章節如何轉換

舊手冊目錄提供了重要線索。新版採用它的訓練功能，但更新內容與工具。

| 舊版功能 | 新版對應 | 調整方向 |
|---|---|---|
| 規則與辦法 | 日常運作 | 改成週報、GitHub、雲端資料夾、會議紀錄與離組交接 |
| 新生暑期工作 | 8 週路線圖 | 改成資料清理、傳統抽水試驗、LDL、轉移不確定性與小型報告 |
| 檔案與程式管理 | 檔案與程式管理 | 加入 reproducibility、source/raw/derived/output 分層 |
| Group Seminar 與 Presentation | 報告訓練 | 強調問題、資料、方法、限制、下一步，而不是只報投影片 |
| 論文撰寫 | 論文與發表 | 保留架構與改稿精神，加入 journal strategy、review response 與 AI 稽核 |
| 科技英文與引用格式 | 寫作工具箱 | 保留 WRR/AGU 等水文期刊習慣，但改成現代引用管理與 Overleaf/GitHub 工作流 |
| 研究相關輔助資料 | 研究訓練 | 更新為 Google Scholar、Web of Science、USGS/open data、Python、Jupyter、AI agent |
| 方法篇與時間管理 | 個人研究系統 | 改成週期性輸出、研究問題拆解、證據庫與專案節奏 |
| 舊式軟體與硬體備忘 | 現代工具 | 只保留「工具要能重現與交接」的原則，不保留過時工具清單 |

## OCR 工作規則

使用 `scripts/gwguide_ocr.py` 時，預設輸出到 `.gwguide-private/ocr`：

```bash
python scripts/gwguide_ocr.py
```

如果只要先測試前幾頁：

```bash
python scripts/gwguide_ocr.py --limit 5
```

OCR 後要檢查：

- `manifest.json`：每張照片的順序、檔名、尺寸、平均信心分數與是否需要人工檢查。
- `pages/*.md`：逐頁 OCR 草稿。
- `combined-private-ocr.md`：合併草稿，只能本機使用。

## 公開文字的標準

放進網站前，每段文字都要符合：

1. 語氣像台灣正體中文使用者，不是英翻中。
2. 不直接引用私人路徑、人名、電話、email 或舊研究室內部安排。
3. 不把舊制度說成目前制度。
4. 不把 OCR 猜測當成原文。
5. 如果是林研究室新增內容，要清楚對齊目前研究主線與工具。

這份手冊可以借用舊手冊的詳細度，但不能把舊手冊的權責、脈絡與私人資訊一起帶進來。
