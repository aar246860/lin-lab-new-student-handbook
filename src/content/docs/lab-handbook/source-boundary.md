# 來源邊界

這份手冊的主要來源是一批舊版地下水研究室手冊照片。公開版的目標不是重新寫一套漂亮的新手冊，而是盡量保留原手冊的章節、條列、須知語氣與訓練密度，只做最小限度修改。

## 三層資料

| 層級 | 位置 | 是否公開 | 用途 |
|---|---|---|---|
| 原始照片 | 本機桌面的 `GW guide` 資料夾 | 否 | 歷史手冊來源，不放入 repo |
| OCR 草稿 | `.gwguide-private/ocr` | 否 | 幫助辨識章節、條列與頁碼，不直接公開 |
| 公開手冊 | `src/content/docs/lab-handbook/` | 可公開 | 原手冊最小修改版，供學生閱讀 |

## 最小限度修改原則

1. 保留原章節編號，例如 1.1、1.2、1.5、2.3、2.5。
2. 保留原本「須知」「要點」「注意事項」的條列格式。
3. 保留舊工具名稱與舊訓練脈絡；必要時只在括號中補目前替代工具。
4. 不把原手冊改寫成行銷文案、課程文案或 AI 風格總結。
5. 不公開私人路徑、聯絡資訊、舊研究室內部權責或不宜公開人名。
6. OCR 明顯錯字須人工修正，不能把亂碼當成原文。
7. 若內容明顯屬於外部書籍或教材摘要，只保留手冊中的使用功能，不整段公開轉載。

## 為什麼不直接貼 OCR

OCR 草稿不是正式文字。它可能混入錯字、背面透字、舊路徑、聯絡資訊、舊制度與排版殘留。公開版要保留原味，但仍需人工整理：

1. 修正 OCR 錯字。
2. 刪除私人資訊。
3. 保留原章節與條列。
4. 用最少文字說明現代工具替代。

## OCR 工作規則

使用 `scripts/gwguide_ocr.py` 時，預設輸出到 `.gwguide-private/ocr`：

```bash
python scripts/gwguide_ocr.py
```

如果只要先測試前幾頁：

```bash
python scripts/gwguide_ocr.py --limit 5
```

OCR 後要檢查：

- `manifest.json`：每張照片的順序、檔名、尺寸、平均信心分數與是否需要人工檢查。
- `pages/*.md`：逐頁 OCR 草稿。
- `combined-private-ocr.md`：合併草稿，只能本機使用。

## 公開文字的標準

放進網站前，每段文字都要符合：

1. 像台灣研究室手冊，不像廣告文案。
2. 能看出來自舊手冊章節，而不是重新發明。
3. 條列要足夠細，不能只剩抽象原則。
4. 可公開、可交接、可讓新生照著做。

這份手冊的方向是「原版最小修改」，不是「概念重寫」。
