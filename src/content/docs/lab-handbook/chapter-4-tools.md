# 第四章 研究相關輔助資料

研究工具不是越多越好，而是要能穩定完成資料整理、分析、繪圖、寫作與交接。本章整理研究室常用工具類別與基本使用要求。

## 4.1 工具使用原則

1. 工具要服務研究問題，不要為了學工具而學工具。
2. 同一個任務盡量固定流程，避免每次換軟體導致結果不可比較。
3. 重要分析要能重跑；只用滑鼠點出的結果，須補上操作紀錄。
4. 圖表與表格須留下原始資料、處理程式與輸出檔。
5. 舊工具若仍能讀取歷史資料，可保留；新分析則優先使用可重現、可版本控制的工具。

## 4.2 文獻搜尋與管理

常用來源包括 Google Scholar、Web of Science、Scopus、期刊網站、學位論文系統與政府公開資料。每篇重要文獻至少記錄：

- 作者、年份、題名、期刊或會議。
- 研究問題。
- 使用資料。
- 方法或模型。
- 主要結果。
- 限制。
- 與自己研究的關係。

文獻 PDF 不要只放在下載資料夾。建議使用 Zotero、EndNote 或 BibTeX 管理，並同步保存筆記。

## 4.3 資料夾結構

每個研究專案至少包含：

```text
project/
  data_raw/
  data_processed/
  code/
  figures/
  tables/
  manuscript/
  notes/
  references/
  outputs/
```

`data_raw` 原則上不修改。所有清理、篩選、插值、轉換與統計結果都應放在 `data_processed` 或 `outputs`，並保留處理程式。

## 4.4 檔名規則

檔名應包含日期、主題、動作與版本。例如：

```text
2026-06-25_pumping-test_drawdown-cleaning_v01.py
2026-06-25_tu-benchmark_k-comparison_v02.png
2026-06-25_manuscript_intro_R1.docx
```

避免使用：

```text
new.docx
final.docx
final_final.docx
figure_new2.png
```

檔名要讓三個月後的自己看得懂。

## 4.5 繪圖工具

繪圖可以使用 Python、MATLAB、R、Excel、Grapher、Surfer、Tecplot、QGIS 或其他工具。無論使用哪個工具，都要符合：

1. 軸標題與單位清楚。
2. 圖例不要遮住資料。
3. 顏色能區分不同資料或模型。
4. 字體大小適合報告或論文。
5. 圖說能解釋圖中重點。
6. 原始資料與繪圖程式可追溯。

若圖要放入論文，優先輸出高解析度圖檔或向量圖。

## 4.6 程式與計算工具

可使用 Python、MATLAB、R、Fortran、Jupyter Notebook 或其他工具。基本要求：

1. 程式開頭寫明用途。
2. input 與 output 檔名清楚。
3. 參數集中管理，不要散在程式各處。
4. 跑完後輸出主要設定與結果摘要。
5. 重要版本要保存，不要直接覆蓋。
6. 若使用 AI 協助寫程式，要自己檢查邏輯與結果。

## 4.7 Word、簡報與 PDF

文書處理常見要求：

- Word 文稿使用標題階層、圖表標號、交叉引用與追蹤修訂。
- PowerPoint 每張投影片只放一個重點，圖要大、字要少。
- PDF 應保留可搜尋文字；掃描檔若可行，先做文字辨識。
- 給老師或同學看的檔案，要避免缺圖、缺字型或連結失效。

## 4.8 備份與交接

重要資料至少要有兩份備份。建議一份在本機，一份在雲端或 GitHub。交接時應包含：

1. 最新文稿。
2. 圖表來源資料。
3. 程式與執行說明。
4. 文獻清單。
5. 未完成問題。
6. 不應再使用的舊版本或錯誤結果說明。
