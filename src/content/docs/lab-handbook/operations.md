# 日常運作

研究室日常運作的目的不是增加行政負擔，而是讓每個人的研究可以被接續、被檢查、被修正。你留下的資料夾、週報、圖、程式與簡報，是未來自己、老師、學長姐與 AI Agent 都會用到的工作記憶。

## 新生暑期工作

新生暑期工作有三個目標：

1. 先建立資料與程式的基本秩序。
2. 走完一個小型地下水資料分析流程。
3. 練習把模型結果翻譯成有邊界的工程判斷。

暑期結束前，至少要留下：

- 一份整理好的資料表。
- 一份可重跑的 Python 或 notebook。
- 一張主圖。
- 一份 5 頁以內的小報告。
- 一段 5 分鐘口頭報告。
- 一份自評：哪些步驟是自己懂的，哪些只是照著做。

## 每週週報

每週交 `weekly-note.md`，格式固定如下：

```text
本週完成：
本週卡住：
我已經試過：
目前證據：
下週要交：
需要老師確認：
```

週報不要寫成心情日記，也不要只寫「繼續看文獻」。如果看文獻，要列出你讀了哪篇、它解決什麼問題、你準備如何用到自己的題目。

## 檔案與程式管理

每個專案至少分成五層：

```text
project/
  README.md
  data_raw/
  data_clean/
  scripts/
  figures/
  manuscript/
  notes/
```

基本規則：

- `data_raw/` 不修改，只保留原始檔。
- `data_clean/` 必須能由程式重建。
- `scripts/` 裡每個程式要有用途說明。
- `figures/` 的圖要能追到產生它的程式與資料。
- `manuscript/` 放文章草稿、回覆審查與投稿版本。
- `notes/` 放研究筆記、會議紀錄與 AI Agent 摘要。

檔名要能看懂，不要使用 `final_final_v3_new.xlsx` 這類名稱。建議格式：

```text
YYYYMMDD_topic_action_version.ext
```

## Group Seminar

Group Seminar 不是表演，而是把研究推進到下一步的工作會議。報告人需要準備：

1. 研究問題：這週到底要解決什麼？
2. 資料來源：資料從哪裡來，是否已清理？
3. 方法：用了哪個解析模型、數值模型或統計方法？
4. 圖與結果：至少一張可以討論的主圖。
5. 限制：目前哪裡還不能下結論？
6. 下週工作：要做什麼才能排除不確定性？

主持人要控制時間、整理問題、確認下一步。安排人要提前確認報告順序與設備，不要當天才處理。

## Presentation

每次報告都要先想聽眾。專題生、研究生、工程顧問、國際學者與期刊 reviewer 需要聽到的重點不同。

基本架構：

1. 背景：為什麼這個問題重要？
2. 缺口：傳統方法哪裡不夠？
3. 方法：你的模型或流程如何處理這個缺口？
4. 結果：圖表要回答問題，不要只是展示跑出來的東西。
5. 邊界：資料、模型與應用限制。
6. 下一步：需要哪些資料或實驗才能讓結論更強？

## 離開或轉題前交接

任何人離開研究室、轉題、畢業或暫停專案前，要交接：

- 最新資料夾位置。
- 可重跑的程式與環境說明。
- 目前最好的一張圖。
- 目前最可靠的文字摘要。
- 未完成問題清單。
- 不應該繼續使用的錯誤版本。

研究室最怕的不是題目失敗，而是沒有人知道資料在哪裡、哪些結果可靠、哪些結論已被推翻。
