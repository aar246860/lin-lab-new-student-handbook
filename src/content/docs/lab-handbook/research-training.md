# 新生暑期工作與研究工具

本頁依舊手冊第 1.2、1.6、1.7 與第 4、5 章目錄重排。原則是先保留舊手冊的訓練順序，再於括號中補上目前可用工具。不是重新發明一套課程。

## 1.2 新生暑期工作

### 1.2.1 先讀資料

暑期開始後，先閱讀下列資料：

1. 研究室手冊。
2. 研究報告格式、科學論文寫作、科技英文寫作、會議簡報相關資料。
3. 讀書方法、時間管理與研究過程中常見問題。
4. 老師指定之研究方向論文。
5. 自己研究題目相關的近期期刊論文。

閱讀後不要求立刻寫長篇心得，但須能說出：這份資料教什麼、與自己的研究有何關係、哪些部分目前看不懂。

### 1.2.2 建立個人工作環境

1. 確認座位、電腦、網路、帳號、共用資料夾與備份位置。
2. 建立個人資料夾，依照研究室檔案命名規則存放。
3. 建立文獻資料庫，至少包含 PDF、筆記與引用資訊。
4. 建立一個練習專案資料夾，包含 `data`、`program`、`figure`、`manuscript`、`note` 等分類。

### 1.2.3 蒐集研究方向資料

1. 蒐集研究方向相關論文、學位論文、技術報告與公開資料。
2. 每篇文獻須記錄作者、年份、題名、研究問題、方法、資料、主要結果與限制。
3. 若文獻與自己題目高度相關，須多寫幾句說明差異與可借用之處。
4. 蒐集資料時要保留來源，不可只留下重新命名後的檔案。

### 1.2.4 學習研究工具

舊手冊列出許多當時常用工具。為保留原手冊脈絡，本版保留工具類別與部分名稱，並在括號內補現代可用工具。

| 類別 | 舊手冊工具脈絡 | 目前可用工具 |

| 文書處理 | Word、PowerPoint、Excel、PDF 工具、輸入法 | Word、PowerPoint、Excel、Overleaf、Markdown |

### 1.2.5 十月起讀書報告

1. 新生十月起開始讀書報告。
2. 報告內容不只是翻譯摘要，而是說明：
   - 研究問題。
   - 作者使用的方法或模型。
   - 主要資料與假設。
   - 重要結果。
   - 優點與限制。
   - 對自己研究的可能用途。
3. 報告後須把投影片、文獻 PDF、筆記與老師或同學的意見放入資料夾。

## 1.7 建議讀書目

本節保留舊手冊的分類，不照抄書籍內容。

### 1.7.1 讀書方法

1. 先看目錄、摘要、圖表與結論。
2. 再看方法與假設。
3. 最後整理可借用之處與限制。
4. 讀完後須留下自己的筆記，不只畫線。

### 1.7.2 中文論文寫作

1. 句子要清楚，主詞與動詞要完整。
2. 一段處理一個主題。
3. 圖表要能支撐文字。
4. 不要用「很多」「一些」「非常重要」等無法檢查的語句取代證據。

### 1.7.3 英文科技論文寫作

1. 先求清楚，再求漂亮。
2. 一句話不要塞太多意思。
3. 方法、結果與討論要分清楚。
4. 文獻回顧要說出前人做了什麼、缺點在哪裡、本文接著處理什麼。

### 1.7.4 時間管理

1. 每週至少留一段完整時間做研究，不只利用零碎時間。
2. 長工作分成可檢查的小項目。
3. 每週會面前整理已完成、卡住、下一步。
4. 假期前整理資料與備份，不把問題留到開學後才想。

### 1.7.5 會議與 Presentation

1. 報告前先想聽眾需要知道什麼。
2. 每張投影片只處理一個重點。
3. 圖要比文字重要。
4. 結尾要留下具體問題、結果或下一步。

## 第 4 章研究相關輔助資料（對應）

舊手冊第 4 章主要是工具與資料來源說明，包括圖書館、期刊、論文資料庫、文獻管理、英文常用句首、標點符號、甘梯圖、繪圖、文書、程式與試算表等。林研究室目前對應如下：

1. 期刊與論文查詢：Google Scholar、Web of Science、Scopus、圖書館資料庫。
2. 文獻管理：Zotero 或 BibTeX。
3. 寫作與排版：Word、Overleaf、Markdown。
4. 圖表：Python、QGIS、Excel、必要時使用舊繪圖工具讀取舊檔。
5. 程式：Python 為主，必要時閱讀或維護 Fortran/MATLAB legacy code。
6. 進度：甘梯圖、issue board、每週進度檔。

## 第 5 章方法篇（對應）

舊手冊第 5 章包括研究初期工作、讀書方法、時間管理、習慣領域與程式寫作。林研究室保留其精神：

1. 研究初期先把問題、資料、方法與可行性講清楚。
2. 不懂的地方先列出來，不要假裝已經懂。
3. 讀書要有輸出，至少要變成筆記、比較表、圖或報告。
4. 時間管理以每週可檢查產出為核心。
5. 程式寫作以可重跑、可交接、可檢查為核心。

## 本研究室新增項目

以下是林研究室新增，不屬於舊手冊原文，但會放入新生訓練：

1. 傳統抽水試驗分析：Theis、Cooper-Jacob、漏水、井儲、skin 與邊界。
2. Lagging Darcy / Lagging Theory：理解 flux-gradient asynchrony，不把 lagging 當單純訊號平移。
3. 轉移不確定性：比較不同合理解析模型造成的參數與決策差異。
4. 決策可靠度：把參數差異傳遞到抽水量、回復時間、failure probability 或安全邊界。
5. AI Agent 稽核：每次 agent 輸出都要回到資料、文獻、程式與 claim 檢查。
