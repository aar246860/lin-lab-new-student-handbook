# 研究訓練

研究訓練的目標不是讓每個新生一開始就做頂尖題目，而是讓你逐步建立能自己判斷的能力。你要知道資料怎麼來、模型假設是什麼、參數怎麼被解釋、結果會如何影響工程決策。

## 舊手冊對應

本頁主要改寫自舊手冊第 1.2 節新生暑期工作、第 1.6 節研究工具自評表、第 1.7 節建議讀書目，以及第 4-5 章研究輔助資料與方法篇。對應 OCR 頁為 006、012、013，並參照目錄頁 003-004。舊手冊的核心精神是：新人先把工具、文獻、寫作與時間管理練起來，再進入真正研究題目。林版把這些工作更新為地下水資料、抽水試驗、Lagging Darcy、轉移不確定性與 AI 輔助研究。

## 八週入門路線

這條路線不是要你八週內做出完整論文，而是讓你通過基本考驗：拿到一筆資料後，能整理、畫圖、選模型、說明假設、判斷限制，最後用五分鐘講清楚。

| 週次 | 主題 | 產出 |
|---|---|---|
| 1 | 資料整理與圖表 | 一張水位或洩降時間序列圖 |
| 2 | Theis 與 Cooper-Jacob | 傳統參數估計與假設說明 |
| 3 | 殘差與模型邊界 | 說明模型在哪些時間段失效 |
| 4 | Lagging Darcy 入門 | normalized response 與非同步訊號示意 |
| 5 | 轉移不確定性 | 比較不同解析模型得到的參數差異 |
| 6 | 決策可靠度 | 把參數差異推到 pumping limit 或 recovery time |
| 7 | 小型專題 | 完成一個可重現分析 |
| 8 | 口頭報告 | 5 分鐘簡報與修正後小報告 |

每週都要留下三種紀錄：資料從哪裡來、程式如何重跑、這週得到的結論支撐到哪裡。若只交出一張漂亮圖，卻無法追到資料與程式，視為未完成。

## 工具自評

舊手冊用自評表要求學生標記「不懂、尚可、熟悉」。林版保留這個做法，但工具改成現代研究流程。每月自評一次：

| 類別 | 不懂 | 尚可 | 熟悉 | 最低標準 |
|---|---:|---:|---:|---|
| 作業系統與檔案管理 |  |  |  | 能整理專案資料夾、備份與命名。 |
| Python / Jupyter |  |  |  | 能讀資料、清理資料、畫圖與輸出表格。 |
| Git / GitHub |  |  |  | 能 commit、push、看 diff，不覆蓋別人的變更。 |
| 文獻管理 |  |  |  | 能用 Zotero 或 BibTeX 管理 PDF、筆記與引用。 |
| 繪圖與排版 |  |  |  | 能產生可投稿等級的圖，標示單位、圖例與資料來源。 |

| 不確定性與可靠度 |  |  |  | 能把模型差異推到參數範圍與決策端點。 |

自評不是拿來評分，而是讓你知道哪個能力會卡住研究。若同一項連續兩個月停在「不懂」，就要排成訓練任務。

## 讀書與文獻蒐集

舊手冊要求新人讀研究方法、中文論文寫作、英文科技寫作、時間管理與會議英語。林版的讀書任務分成五類：

1. 地下水基礎：抽水試驗、含水層參數、邊界、井效應與尺度問題。
2. 寫作方法：摘要、前言、文獻回顧、方法再現性、結果與結論。
3. 研究方法：如何拆問題、如何設計案例、如何判斷資料能不能支持 claim。
4. 不確定性：模型選擇、equifinality、轉移不確定性、model factor 與可靠度。
5. AI 工作流：如何用 agent 做文獻掃描、程式重跑、圖表檢查與論文殘留語句稽核。

每讀一篇重要文獻，都要留下四句話：

```text
這篇處理什麼問題：
使用什麼模型或資料：
真正貢獻是什麼：
它對我的題目有什麼用或限制：
```

不要只把 PDF 存起來。文獻如果沒有變成筆記、圖表、比較表或研究判斷，就還沒有進入你的研究系統。

## 傳統水井力學

傳統抽水試驗分析是研究室共同語言。你至少要能說明：

- Theis 解的基本假設。
- Cooper-Jacob late-time approximation 為什麼常用，也在哪裡可能誤導。
- Hantush 類漏水模型與 Neuman 類自由含水層延遲釋水模型想捕捉什麼現象。
- 井儲、skin、邊界與非均質性可能如何改變曲線。

你不需要一開始就推導所有方程式，但不能把軟體輸出的參數當成沒有假設的真值。

### 傳統方法練習

每位新生至少做一次完整傳統分析：

1. 畫 drawdown-time 與 recovery-time 曲線。
2. 用 Theis 或 Cooper-Jacob 得到第一組參數。
3. 標出 early-time、middle-time、late-time 可能對應的機制。
4. 寫出假設：定率抽水、均質、等厚、無限域、井儲是否忽略。
5. 說明哪一段資料不能被傳統模型好好解釋。

這個練習的目的不是貶低傳統方法，而是讓你知道傳統方法何時足夠、何時會把未建模機制塞進參數。

## Lagging Darcy / Lagging Theory

Lagging Darcy 的核心不是單純把訊號平移，而是處理 flux 與 hydraulic gradient 在非穩態條件下可能不同步的水力響應。這種不同步可能來自微觀孔隙結構、流徑繞行、雙孔隙或多尺度交換、流固耦合、邊界響應或其他尚未被顯式建模的機制。

入門時先掌握三個概念：

1. 傳統 Darcy law 是宏觀經驗關係，不等於所有非穩態條件下都瞬時成立。
2. lagging 參數不是萬用材料常數，而是特定資料、尺度與模型條件下的診斷量。
3. 若 lagging 模型只是多參數擬合，研究貢獻會很弱；必須說明它如何改善殘差結構、參數穩定性或工程決策。

新生不需要一開始就宣稱 Lagging Darcy 是所有問題的答案。正確做法是先問：

- 傳統模型殘差是否有系統性，而不是隨機雜訊？
- lagging 參數是否在合理資料視窗內可辨識？
- 加入 lagging 後，K、S 或其他參數是否更穩定？
- 結果是否改變 pumping limit、recovery time 或風險邊界？

## 轉移不確定性

抽水試驗量到的是觀測響應，模型輸出的是參數解釋。中間有一層轉換：

```text
觀測資料 → 解析模型 → 參數估計 → 工程設計值 → 風險判斷
```

轉移不確定性關心的是：同一批觀測資料，若用不同合理模型分析，最後得到的參數與決策會差多少。

研究室訓練時，要避免只問「哪個模型 fit 最好」。更重要的是：

- 哪些模型在物理上可接受？
- 哪些時間段支撐了哪些參數？
- 模型差異是否改變設計結論？
- 能否形成 model factor 或 COV，供可靠度分析使用？

這裡要特別注意：轉移不確定性不是說「所有模型都一樣好」，而是要求你把觀測資料到工程參數之間的模型選擇風險量化出來。若某個模型沒有足夠物理或資料支持，就不能只是因為 fitting 好而被納入。

## 決策可靠度

決策可靠度把模型結果推到工程語言。常見端點包括：

- 抽水量上限。
- 許可抽水時間。
- 回復時間。
- failure probability。
- 需要保留的安全邊界。

學生要練習把「參數差異」翻譯成「決策差異」。如果不同模型得到的水力傳導係數差兩倍，但 pumping limit 幾乎不變，工程意義就有限；如果參數差異讓允許抽水量或風險邊界大幅改變，這就是需要被管理的問題。

## AI 與資料庫

未來研究會逐漸走向大規模資料重分析。USGS、國際抽水試驗資料庫、現地監測資料、重力或地球物理推估結果，都可能被納入比較。AI Agent 可以協助建立資料 registry、清理 metadata、批次跑模型與產生稽核表。

但 AI 不能跳過地下水判斷。任何自動化分析都要回答：

1. 這個資料能不能支持這個模型？
2. 這個模型的參數是否可辨識？
3. 這個結果是否只是補償未建模機制？
4. 這個結論是否會影響決策？

AI Agent 不能取代你的水文判斷。它可以幫你跑更多比較、檢查更多文件、產生更多草稿，但最後每個模型、參數與 claim 都要回到資料、方程式與工程意義。
