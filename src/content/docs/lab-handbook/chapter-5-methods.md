# 第五章 方法篇

研究能力不是只靠聰明，而是靠穩定的方法。遇到困難時，先把問題拆開、留下證據、提出下一步，而不是停在「我不會」。

## 5.1 研究過程常見問題

新生常見問題包括：

| 問題 | 改善方式 |
|---|---|
| 題目太大 | 先縮成一個可驗證的小問題。 |
| 文獻看不懂 | 先抓研究問題、方法、圖表與結論，不必第一遍就懂全部推導。 |
| 資料很多但不知道怎麼用 | 先整理欄位、單位、時間、空間位置與缺漏值。 |
| 模型跑不出來 | 先用最小案例測試，再逐步增加複雜度。 |
| 結果不好解釋 | 回到資料、假設與模型邊界，不要急著寫結論。 |
| 進度說不清楚 | 每週固定寫進度紀錄，讓問題可追蹤。 |

## 5.2 研究初期工作

研究初期不要急著做大模型。先完成：

1. 定義問題：你要回答什麼？
2. 蒐集文獻：誰做過類似問題？
3. 整理資料：有哪些資料可用？品質如何？
4. 建立基準方法：傳統方法能做到什麼程度？
5. 找出限制：目前方法哪裡不夠？
6. 設計小實驗：用最小案例確認想法是否站得住。

一開始做小，不代表研究小；小案例若能清楚說明問題，反而更容易推進。

## 5.3 讀書方法

讀論文可以分三遍：

1. 第一遍：看題名、摘要、圖表與結論，判斷是否相關。
2. 第二遍：看前言、方法與主要圖表，整理研究問題與方法。
3. 第三遍：看推導、參數、假設、限制與可借用之處。

每篇重要文獻至少留下五句話：

```text
這篇在解決什麼問題：
作者用了什麼方法：
最重要的結果是：
主要限制是：
和我的研究有什麼關係：
```

不要只畫重點線。畫線不等於理解。

## 5.4 時間管理

研究工作常被短任務切碎，所以要固定安排：

- 文獻閱讀時間。
- 資料整理時間。
- 程式與分析時間。
- 寫作時間。
- 每週回顧時間。

每天結束前，寫下明天第一件要做的事。每週結束前，整理本週完成項目、未完成原因與下週目標。

## 5.5 問題拆解

遇到問題時，先用下面格式整理：

| 問題類型 | 你要確認什麼 |
|---|---|
| 資料問題 | 資料來源、單位、缺漏、異常值、座標與時間。 |
| 程式問題 | input、output、錯誤訊息、最小可重現案例。 |
| 模型問題 | 假設、參數、邊界條件、初始條件與適用範圍。 |
| 寫作問題 | 研究問題是否清楚、段落是否有主旨、證據是否足夠。 |
| 簡報問題 | 聽眾是誰、要帶走什麼訊息、哪張圖最重要。 |

把問題拆小，才知道下一步要問誰、查什麼、算什麼。

## 5.6 研究討論準備

每次和老師或同學討論前，請準備：

1. 本週完成項目。
2. 目前最重要的圖或表。
3. 遇到的問題。
4. 你已經嘗試過的方法。
5. 下週可執行的兩到三個選項。

討論不是報流水帳，而是讓大家幫你做下一步判斷。

## 5.7 自我檢查

每兩週檢查一次：

- 我的資料是否可追溯？
- 我的程式是否可重跑？
- 我的圖是否能說明一個明確訊息？
- 我的文獻筆記是否能支持前言與討論？
- 我的研究問題是否比兩週前更清楚？
- 我的下一步是否具體到明天可以開始做？

若答案多數是否定，代表要先整理，不要繼續堆新東西。
