# 第二章 論文撰寫

本章依舊手冊第 2 章重排。原手冊中屬於外部書籍或教材的長段內容不逐字公開；其餘研究室寫作規則、流程與提醒，以最小限度修改方式保留。

## 2.1 撰寫進度與校稿時間

論文撰寫須提早規劃，不可等到口試或投稿前才開始。舊手冊的精神是：初稿、校稿、摘要、圖表、送印、寄送口委或投稿都要排入時程。

### 2.1.1 學位論文時程

1. 口試前至少六至八週，應完成第一版完整初稿。
2. 第一版初稿須包含：目錄、摘要草稿、主要章節、圖、表、參考文獻與附錄骨架。
3. 交給老師修改時，應使用追蹤修訂或清楚標記修改處。
4. 第一稿通常需要較長時間修改；第二稿、第三稿才逐步縮短。
5. 口試前一週須完成寄送口委之版本，不宜再大量新增分析。
6. 若論文將改寫成期刊論文，初稿須再提前完成。

### 2.1.2 改稿檔名

1. 給老師修改之文稿，檔名後依修改順序加 a、b、c 或日期。
2. 定稿時移除暫時字尾，保留清楚版本。
3. 投稿後修改稿可用 R1、R2 等標示。
4. 每一輪修改須保存 clean manuscript、marked manuscript 與 change log。

## 2.2 論文大綱／大網架構

### 2.2.1 參考書籍與資料

撰寫論文前，須先讀論文寫作、研究報告格式、科技英文與本手冊相關章節。這些資料的作用不是讓學生照抄句子，而是讓學生知道論文結構、句子、圖表、引用與校稿的基本規矩。

### 2.2.2 中文論文大綱

一般中文論文主要內容包括：

1. 研究動機、目的及問題簡述。
2. 解決方法。
3. 案例研究或資料分析。
4. 結果與討論。
5. 結論與建議。

### 2.2.3 英文論文大綱

英文期刊論文通常包含下列章節，實際名稱可依期刊與研究內容調整：

1. Introduction
   - Background
   - Problem statement
   - Literature review
   - Objectives
   - Purpose and scope
2. Methods / Theory / Methodology / Model / Governing Equations
   - Model verification
   - Study area
   - Data set
   - Application or examples
3. Results / Simulation Results / Data Analysis
   - Case results
   - Sensitivity analysis
   - Comparison
4. Discussion
   - 若 Results 與 Discussion 篇幅不多，可合併為 Results and Discussion。
5. Summary and Conclusions / Conclusions / Concluding Remarks
6. Appendix
7. Notation
8. Acknowledgment
9. References

## 2.3 論文撰寫內容

### 2.3.1 摘要

1. 摘要須簡明說明問題、方法、資料或案例、主要結果與意義。
2. 摘要內應避免方程式、方程式編號、文獻引用與過多參數符號。
3. 摘要不是前言縮短版，也不是結論逐字重複。

### 2.3.2 關鍵字

1. 關鍵字應能幫助讀者與資料庫找到論文。
2. 題名中已出現的名詞，不一定需要再當關鍵字。
3. 地下水相關論文可視情況包含 groundwater、aquifer test、pumping test、well hydraulics、uncertainty 等字詞。

### 2.3.3 符號表

1. 文中若使用許多符號或變數，宜備符號表。
2. 同一篇論文內，同一個變數不可有兩個不同定義。
3. 符號表通常放在圖表目錄之後。
4. 初稿完成後，須對照符號表檢查是否有重複、漏定義或前後不一致。

### 2.3.4 前言

前言一般包括三部分：一般說明、問題所在及論文目的。寫作時注意：

1. 說明研究背景時，不要扯太遠。
2. 前言須交代前人研究已做到什麼程度，尚有何缺點或限制，因此引發本文研究動機。
3. 前言中避免出現過多方程式。
4. 前言通常在點出問題後接文獻回顧。
5. 前言須清楚定義工程問題或科學問題的範圍。

### 2.3.5 文獻回顧

1. 一般可依發表年代敘述；若有不同 topic 或 issue，應分段回顧。
2. 同一段落內的文獻須屬於相同主題。
3. 不直接相關的文獻，用一兩句說明即可。
4. 與本文非常相關的文獻，宜多寫一些，包括其方法、結果、限制及與本文不同處。
5. 文獻回顧不可只是列作者名字，須說明作者處理什麼問題、用什麼方法、得到什麼結果。
6. 引用文獻內容可以改寫與摘要，但不可照抄相同語句。

### 2.3.6 理論推導或研究方法

此章節是研究精華所在，包含理論推導與分析方法。應注意：

1. 論文重要價值之一是再現性。讀者照著文中方法，應可得到相同結果。
2. 重要細節須說明清楚，無關整體但冗長之推導可列在附錄。
3. 若重要概念或理論不是自己推導，須引用來源。
4. 可依研究需要建立自己的章節、方程式與變數，不必完全沿用相近文獻格式。
5. 書籍或文獻已有之內容，不宜大量引用，只摘其重點或相關方程式並說明來源。
6. 單位原則上使用公制。

### 2.3.7 結果與討論

1. 結果與討論是論文核心。須描述推導、實驗、模擬或資料分析結果。
2. 少用形容詞，盡量用圖、表、數值與比較說明。
3. 在描述個別案例之前，先說明案例設計目的與案例數目。
4. 與前人結果比較時，重點在說明自己方法的優點與適用條件，避免只批評別人缺點。
5. 重要數據與關鍵結果應放在本文內，不可全部推到附錄。

### 2.3.8 結論

1. 結論須提出具體、明確成果。
2. Summary 是整理主要結果；Conclusions 須解釋結果意義。
3. 好的結論須回答：新的發現是什麼？有何意義？為何結果成立？限制為何？
4. 結論中須清楚描述本文重要價值或貢獻。
5. 結論內不宜再提新的圖或表。

### 2.3.9 致謝、附錄與參考文獻

1. 致謝須依實際經費、計畫、資料與協助來源撰寫。
2. 附錄可放較長數學推導、證明、補充資料或程式說明，但主結果仍須在本文中討論。
3. 附錄方程式可依 A1、A2 或 B1、B2 等方式編號。
4. 參考文獻只列本文引用過、且作者真正讀過的文獻。
5. References 中作者、年代、題名、期刊、卷期、頁碼與 DOI 須一致。

## 2.4 論文撰寫次序

### 2.4.1 下筆次序

1. 先訂論文題目與目錄，之後可再修改。
2. 先把章節列出來，再逐節寫下主要內容。
3. 完成題目與目錄後，先與老師討論，以免寫入不必要內容。
4. 建議撰寫次序可為：摘要、結論、方法、結果與討論、前言。完成前言後，再回頭細改摘要與結論。
5. 補上致謝、參考文獻、圖、表與附錄後，才算完成初稿。

### 2.4.2 拓展研究成果的次序

研究成果可依下列順序逐步整理：

1. 研究計畫或研究構想。
2. 研討會論文或口頭報告。
3. 學位論文。
4. 期刊論文。

### 2.4.3 論文修改的問題

口試、共同作者或審稿意見若要求新增案例、分析或內容，不能只把新增部分貼上去。全文相關描述、摘要、方法、結果、圖表、結論與限制都須同步修改。

## 2.5 論文撰寫細節

### 2.5.1 論文標題與章節標題

1. 論文標題須簡明、扼要、切題。
2. 章節標題避免過長。
3. 章或節標題中盡量不要放變數名。
4. 標題若落在頁面最後一兩行，排版時須調整。

### 2.5.2 撰寫科技英文句子應注意事項

1. 學長姊或自己以前的論文語句，不可以照抄，需改寫。
2. 使用短句，避免長句。一句話只陳述一個主要內容。
3. 句子要有主詞與動詞，避免以很長的分詞片語當主詞。
4. 文章開頭不要使用太多轉折語。
5. 避免過度使用 in this study 等空泛起頭。
6. 文章時態須一致。
7. 重點或結論放在句首，原因與說明放在句尾。
8. 避免使用感性或不定量形容詞，如「一些」「某些」「非常」。
9. 同一句內避免重複使用相同文字或語句。
10. 同一概念在全文中用詞須一致。

### 2.5.3 論文格式

1. 變數用斜體；下標與單位依期刊規定處理。
2. 非公制單位須補公制單位。
3. 縮寫第一次出現時須先寫全名，再於括號中列縮寫。
4. 縮寫定義後，後文須一致使用。
5. 文中提到他人方程式或表時，應以作者、年代與表號或方程式說明，不應讓讀者誤以為是本文內容。
6. 圖、表、附錄、符號、參考文獻與頁碼須一致。

### 2.5.4 完稿前的論文編排

1. 初稿最好一併附參考文獻、圖、表、符號說明及附錄。
2. 文稿宜加頁碼，以免改稿時混淆。
3. 第二次交稿時，須給圖、表與附錄標題。
4. 完稿後，須自己重讀兩次以上，或先與同學互換看稿。
5. 最後須檢查拼字、文法、圖表編號、引用格式與殘留註解。

### 2.5.5 常犯錯誤

1. 拼字錯誤。
2. 縮寫未定義或定義後前後不一致。
3. 變數、參數或模型名稱前後不一致。
4. 圖表沒有單位或說明不足。
5. 文獻引用沒有支撐當前句子。
6. 把未驗證推論寫成已證明結論。
7. AI 改稿後殘留自問自答、revision note、placeholder 或不屬於正式文稿的句子。

## 2.10-2.12 研討會與期刊論文

1. 研討會論文不是期刊論文的縮小版，重點在清楚表達問題、方法、初步結果與後續發展。
2. 期刊論文須在投稿前確認目標期刊、讀者、格式、創新、比較對象與資料支撐程度。
3. 給老師修改研討會或期刊論文時，須附上乾淨稿、修改稿、圖表、資料來源與需要老師判斷的問題。
4. 若被拒稿，先分類拒稿原因：題目定位、創新不足、方法不清、比較不足、資料不足、寫作不清或期刊不合。
