# 第二章 論文撰寫

本章依舊手冊第 2 章重排。原手冊中屬於外部書籍或教材的長段內容不逐字公開；其餘研究室寫作規則、流程與提醒，以最小限度修改方式保留。

## 2.1 撰寫進度與校稿時間

論文撰寫須提早規劃，不可等到口試或投稿前才開始。舊手冊的精神是：初稿、校稿、摘要、圖表、送印、寄送口委或投稿都要排入時程。

### 2.1.1 學位論文時程

1. 口試前至少六至八週，應完成第一版完整初稿。
2. 第一版初稿須包含：目錄、摘要草稿、主要章節、圖、表、參考文獻與附錄骨架。
3. 交給老師修改時，應使用追蹤修訂或清楚標記修改處。
4. 第一稿通常需要較長時間修改；第二稿、第三稿才逐步縮短。
5. 口試前一週須完成寄送口委之版本，不宜再大量新增分析。
6. 若論文將改寫成期刊論文，初稿須再提前完成。

### 2.1.2 改稿檔名

1. 給老師修改之文稿，檔名後依修改順序加 a、b、c 或日期。
2. 定稿時移除暫時字尾，保留清楚版本。
3. 投稿後修改稿可用 R1、R2 等標示。
4. 每一輪修改須保存 clean manuscript、marked manuscript 與 change log。

## 2.2 論文大綱／大網架構

### 2.2.1 參考書籍與資料

撰寫論文前，須先讀論文寫作、研究報告格式、科技英文與本手冊相關章節。這些資料的作用不是讓學生照抄句子，而是讓學生知道論文結構、句子、圖表、引用與校稿的基本規矩。

### 2.2.2 中文論文大綱

一般中文論文主要內容包括：

1. 研究動機、目的及問題簡述。
2. 解決方法。
3. 案例研究或資料分析。
4. 結果與討論。
5. 結論與建議。

### 2.2.3 英文論文大綱

英文期刊論文通常包含下列章節，實際名稱可依期刊與研究內容調整：

1. Introduction
   - Background
   - Problem statement
   - Literature review
   - Objectives
   - Purpose and scope
2. Methods / Theory / Methodology / Model / Governing Equations
   - Model verification
   - Study area
   - Data set
   - Application or examples
3. Results / Simulation Results / Data Analysis
   - Case results
   - Sensitivity analysis
   - Comparison
4. Discussion
   - 若 Results 與 Discussion 篇幅不多，可合併為 Results and Discussion。
5. Summary and Conclusions / Conclusions / Concluding Remarks
6. Appendix
7. Notation
8. Acknowledgment
9. References

## 2.3 論文撰寫內容

### 2.3.1 摘要

1. 摘要須簡明說明問題、方法、資料或案例、主要結果與意義。
2. 摘要內應避免方程式、方程式編號、文獻引用與過多參數符號。
3. 摘要不是前言縮短版，也不是結論逐字重複。

### 2.3.2 關鍵字

1. 關鍵字應能幫助讀者與資料庫找到論文。
2. 題名中已出現的名詞，不一定需要再當關鍵字。
3. 地下水相關論文可視情況包含 groundwater、aquifer test、pumping test、well hydraulics、uncertainty 等字詞。

### 2.3.3 符號表

1. 文中若使用許多符號或變數，宜備符號表。
2. 同一篇論文內，同一個變數不可有兩個不同定義。
3. 符號表通常放在圖表目錄之後。
4. 初稿完成後，須對照符號表檢查是否有重複、漏定義或前後不一致。

### 2.3.4 前言

前言一般包括三部分：一般說明、問題所在及論文目的。寫作時注意：

1. 說明研究背景時，不要扯太遠。
2. 前言須交代前人研究已做到什麼程度，尚有何缺點或限制，因此引發本文研究動機。
3. 前言中避免出現過多方程式。
4. 前言通常在點出問題後接文獻回顧。
5. 前言須清楚定義工程問題或科學問題的範圍。

### 2.3.5 文獻回顧

1. 一般可依發表年代敘述；若有不同 topic 或 issue，應分段回顧。
2. 同一段落內的文獻須屬於相同主題。
3. 不直接相關的文獻，用一兩句說明即可。
4. 與本文非常相關的文獻，宜多寫一些，包括其方法、結果、限制及與本文不同處。
5. 文獻回顧不可只是列作者名字，須說明作者處理什麼問題、用什麼方法、得到什麼結果。
6. 引用文獻內容可以改寫與摘要，但不可照抄相同語句。

### 2.3.6 理論推導或研究方法

此章節是研究精華所在，包含理論推導與分析方法。應注意：

1. 論文重要價值之一是再現性。讀者照著文中方法，應可得到相同結果。
2. 重要細節須說明清楚，無關整體但冗長之推導可列在附錄。
3. 若重要概念或理論不是自己推導，須引用來源。
4. 可依研究需要建立自己的章節、方程式與變數，不必完全沿用相近文獻格式。
5. 書籍或文獻已有之內容，不宜大量引用，只摘其重點或相關方程式並說明來源。
6. 單位原則上使用公制。

### 2.3.7 結果與討論

1. 結果與討論是論文核心。須描述推導、實驗、模擬或資料分析結果。
2. 少用形容詞，盡量用圖、表、數值與比較說明。
3. 在描述個別案例之前，先說明案例設計目的與案例數目。
4. 與前人結果比較時，重點在說明自己方法的優點與適用條件，避免只批評別人缺點。
5. 重要數據與關鍵結果應放在本文內，不可全部推到附錄。

### 2.3.8 結論

1. 結論須提出具體、明確成果。
2. Summary 是整理主要結果；Conclusions 須解釋結果意義。
3. 好的結論須回答：新的發現是什麼？有何意義？為何結果成立？限制為何？
4. 結論中須清楚描述本文重要價值或貢獻。
5. 結論內不宜再提新的圖或表。

### 2.3.9 致謝、附錄與參考文獻

1. 致謝須依實際經費、計畫、資料與協助來源撰寫。
2. 附錄可放較長數學推導、證明、補充資料或程式說明，但主結果仍須在本文中討論。
3. 附錄方程式可依 A1、A2 或 B1、B2 等方式編號。
4. 參考文獻只列本文引用過、且作者真正讀過的文獻。
5. References 中作者、年代、題名、期刊、卷期、頁碼與 DOI 須一致。

## 2.4 論文撰寫次序

### 2.4.1 下筆次序

1. 先訂論文題目與目錄，之後可再修改。
2. 先把章節列出來，再逐節寫下主要內容。
3. 完成題目與目錄後，先與老師討論，以免寫入不必要內容。
4. 建議撰寫次序可為：摘要、結論、方法、結果與討論、前言。完成前言後，再回頭細改摘要與結論。
5. 補上致謝、參考文獻、圖、表與附錄後，才算完成初稿。

### 2.4.2 拓展研究成果的次序

研究成果可依下列順序逐步整理：

1. 研究計畫或研究構想。
2. 研討會論文或口頭報告。
3. 學位論文。
4. 期刊論文。

### 2.4.3 論文修改的問題

口試、共同作者或審稿意見若要求新增案例、分析或內容，不能只把新增部分貼上去。全文相關描述、摘要、方法、結果、圖表、結論與限制都須同步修改。

## 2.5 論文撰寫細節

### 2.5.1 論文標題與章節標題

1. 論文標題須簡明、扼要、切題。
2. 章節標題避免過長。
3. 章或節標題中盡量不要放變數名。
4. 標題若落在頁面最後一兩行，排版時須調整。

### 2.5.2 撰寫科技英文句子應注意事項

1. 學長姊或自己以前的論文語句，不可以照抄，需改寫。
2. 使用短句，避免長句。一句話只陳述一個主要內容。
3. 句子要有主詞與動詞，避免以很長的分詞片語當主詞。
4. 文章開頭不要使用太多轉折語。
5. 避免過度使用 in this study 等空泛起頭。
6. 文章時態須一致。
7. 重點或結論放在句首，原因與說明放在句尾。
8. 避免使用感性或不定量形容詞，如「一些」「某些」「非常」。
9. 同一句內避免重複使用相同文字或語句。
10. 同一概念在全文中用詞須一致。

### 2.5.3 論文格式

1. 變數用斜體；下標與單位依期刊規定處理。
2. 非公制單位須補公制單位。
3. 縮寫第一次出現時須先寫全名，再於括號中列縮寫。
4. 縮寫定義後，後文須一致使用。
5. 文中提到他人方程式或表時，應以作者、年代與表號或方程式說明，不應讓讀者誤以為是本文內容。
6. 圖、表、附錄、符號、參考文獻與頁碼須一致。

### 2.5.4 完稿前的論文編排

1. 初稿最好一併附參考文獻、圖、表、符號說明及附錄。
2. 文稿宜加頁碼，以免改稿時混淆。
3. 第二次交稿時，須給圖、表與附錄標題。
4. 完稿後，須自己重讀兩次以上，或先與同學互換看稿。
5. 最後須檢查拼字、文法、圖表編號、引用格式與殘留註解。

### 2.5.5 常犯錯誤

1. 拼字錯誤。
2. 縮寫未定義或定義後前後不一致。
3. 變數、參數或模型名稱前後不一致。
4. 圖表沒有單位或說明不足。
5. 文獻引用沒有支撐當前句子。
6. 把未驗證推論寫成已證明結論。
7. AI 改稿後殘留自問自答、revision note、placeholder 或不屬於正式文稿的句子。

### 2.5.6 長方程式的縮減

1. 方程式若很簡單，可直接寫在本文內；若式子較長或需要編號，應獨立成列式。
2. Equation number 須靠右，文中再次提到時寫成 Eq. (3) 或 (3)，全文須一致。
3. 每一個方程式代表的意義，以及方程式之間的推導關係，最好都能以簡短文字說明。
4. 英文稿中可用 “may be expressed as” 表示「可表示為」；避免把語氣寫得過度武斷。
5. 方程式下方的變數說明以 where 起頭，where 不內縮且字首小寫。
6. 若連續列出多個方程式，方程式之間須說明各自代表的物理意義；若兩個方程式並列，可用 and 連接。
7. 方程式內變數很多時，須建立符號表，並檢查同一符號是否被重複用於不同定義。
8. 若研究方法來自不同文獻，常會遇到相同符號代表不同變數；此時須主動改符號，避免全文混淆。
9. 若方程式很長且重複出現相同長式，應定義新的中間變數或函數，讓主式呈現主要物理結構。
10. 縮減長方程式時，不是把細節藏掉，而是讓讀者看得出主要外力、邊界條件或機制各自貢獻哪些項。

### 2.5.7 序列組成

1. 序列中的指標須寫清楚，例如 for n = 1, 2, ... 或 for n = 1, 2, ..., r - 2。
2. 若在文章內提到迭代次數或級數項，須明確說明 first and second iterations、n = 1, 2, 3, ... 等條件。
3. 若序列只在偶數或奇數條件下成立，須在同一句或下一句說明限制條件。
4. 若序列寫在句尾，省略號後仍須補上句點；若有最大項或終止項，須依期刊格式完整列出。

### 2.5.8 圖表

1. 條列式推理、計算步驟或方法流程若較長或不易懂，應輔以流程圖說明。
2. 計算所得資料與結果，盡量以圖表示出。工程與水文論文應讓他人能查證、引用或重現。
3. 圖表標題不要太長，力求精簡、清楚、可獨立理解。
4. 若文稿篇幅少，圖表可放在第一次提及的段落後；若圖表很多，則依期刊規定放在 References 後或另行上傳。
5. 文章中提到圖或表時，不能只寫「如圖所示」。必須說明曲線、數字或符號代表的重點、意義與特殊處。
6. 圖表應配合本文分析，說明與前人研究或不同案例的差異，不可只作展示。
7. 引用或改繪他人圖表時，須確認著作權與期刊規定；需要授權時必須取得同意。
8. 圖、表引用格式須一致，例如 Fig. 1、Table 2。若在句首提到圖，通常寫 Figure 1，不寫 Fig. 1。
9. Fig. 與 Table 前面通常不加冠詞 the，除非目標期刊另有慣例。
10. 多張圖之間若要比較，須讓座標軸、單位、圖例與顏色能互相對照。

## 2.8 參考文獻格式與 WRR/AGU 類期刊

原手冊特別列出 AGU Publications（WRR format）的參考文獻元素。新版原則如下：

1. 參考文獻格式不可憑印象整理，須以目標期刊最新 Author Guide 或 reference style 為準。
2. Journal article 通常至少包含作者、年代、文章題名、期刊名、卷、期、頁碼或 article number、DOI。
3. Book 或 book chapter 須包含作者或編者、年代、書名或章名、系列名、頁碼、出版商與出版地。
4. Presented paper 須包含作者、年代、論文題名、會議名稱、主辦單位、地點與日期。
5. Report、map、thesis 或 dissertation 須包含作者、年代、題名、報告類型或學位、頁數、發行單位或學校與地點。
6. References 中作者、年代、標題大小寫、期刊縮寫、斜體、標點、DOI 格式須逐筆檢查，不可只靠引用軟體輸出。

## 2.9 學位論文或計畫報告校稿、改稿及印製注意事項

### 2.9.1 自行校稿改稿

1. 中文稿注意標點符號使用；英文稿注意單複數、冠詞、時態與主詞動詞一致。
2. 圖、表與方程式編號須從頭檢查，確認沒有重複或遺漏。
3. 文中引用作者超過三人時，可依期刊規定寫成 Lin et al.；文末 References 則須按格式列出完整作者資訊。
4. 書籍類參考文獻須包含作者、書名、出版商、出版地與年代。
5. 改完稿的新版本不要立刻交出。先和老師修改稿比對，確認每一個追蹤修訂、註解或要求都已處理，再從頭細看一次 typo。
6. 附錄若很多，須先決定附錄編碼方式，例如 Appendix A、Appendix B 或 A1、A2，全文一致。
7. 改稿完成後，老師提供的書籍、資料、文獻或參考檔案須歸還或整理回共享位置，方便下一輪改稿。
8. 正文頁碼、中英文摘要、圖表目錄與附錄頁碼須分清楚。學位論文通常前置頁用 i、ii、iii，正文從 1 開始。
9. 每一輪交稿至少保留 clean manuscript、marked manuscript、response/change log 三份，避免只剩無法追蹤的單一檔案。

### 2.9.2 印製與定稿

1. 若他人資料、儀器手冊或程式說明要作為附錄，須整理成可閱讀格式並補頁碼。
2. 英文稿第一版與最後定稿前都要做 spelling check，但拼字檢查不能取代人工校稿。
3. 若學校或期刊要求 cover、Vita、聲明頁或其他前置頁，須在最後一輪一起檢查。
4. 印製或送出前，先用 PDF 檢查整頁排版：頁碼、圖表、方程式、頁首頁尾、目錄與參考文獻是否完整。
5. 舊版手冊中的紙本封面、書背與顏色規定，現在依學校最新規定處理；不要沿用舊格式。

## 2.10 研討會論文撰寫

### 2.10.1 摘要

1. 看到徵文啟事後，先寫摘要，說明最近研究成果或研究構想，以及可能用途與貢獻。
2. 英文摘要或全文若投稿國內研討會，文前依規定加上中文標題與作者姓名。
3. 摘要送出後，須確認主辦單位是否收到，不要等到截止後才發現沒有完成投稿。
4. 收到接受函後，應儘快完成研究工作，注意全文截止日期；最好在截止日期前兩週完成全文初稿。

### 2.10.2 全文

1. 全文中的摘要不能直接沿用先前投稿摘要。先前摘要多半是預測性描述；全文摘要應改寫成實際成果與貢獻，語氣通常使用過去式或完成式。
2. 寫作前先查看主辦單位提供的格式樣本，盡量按規定格式撰寫初稿，避免完稿後才花大量時間調整格式。
3. 修改完後，自己須將全文細讀一遍，尤其要校對被修改過的段落。
4. 研討會摘要通常先寫題名、作者姓名、服務單位，再寫摘要與內容。
5. 研討會論文題名不必把所有細節都塞進去；完整且最有力的題名可留給學位論文或期刊論文。
6. 若同一研究將發展成期刊論文，研討會文重點可放在新方法、初步結果或特定案例，不要把未完成的推論寫成定論。
7. 文稿送出時加 cover letter 或系統說明；若是學生主責投稿，須同步把送出版本給老師留存。

## 2.11 期刊論文撰寫與投寄

### 2.11.1 期刊論文撰寫

1. 期刊論文通常由研討會論文發展，或由學位論文濃縮而成；但不能只是剪貼縮短，內容必須重新加強。
2. 投稿前一定要閱讀目標期刊的 Author Guide 或 Information for Contributors。格式不符、字數超過規定或圖表不合規，可能在主編初篩階段就被退稿。
3. 撰寫時手邊至少要有目標期刊最近文章一篇，以及自己研究室過去投寄成功或格式接近的文稿一篇作為樣本。
4. 有些期刊限制題名字數或摘要字數，例如 Groundwater 或 WRR 類期刊常有明確限制；實際限制以最新 Author Guide 為準。
5. 特別注意 References 格式：作者、年代、標題、期刊名、卷期、頁碼、DOI 的順序與標點要完全符合期刊規定。
6. Acknowledgments 須依實際情況感謝經費、資料、設備與協助。修改稿若經審查後接受再修，可在適當位置感謝 reviewers 的建議。
7. 最後一稿送出前，用整頁模式逐頁檢查：英文拼字與文法、margin、paper size、equation numbers、章節標題是否落在頁面最後一行、Figure Captions、文中引用文獻是否都列在 References。
8. 被拒稿後，不要只換期刊重投。先分類原因：題目定位、創新不足、方法不清、比較不足、資料不足、寫作不清、期刊不合或格式問題。

### 2.11.2 期刊論文格式

投稿期刊論文一定要按照目標期刊規定格式。至少須檢查下列項目：

1. 題名、摘要、全文、圖表與 supplementary materials 是否有字數、頁數或圖表數限制。
2. 圖表是否需要與本文分開上傳，檔案格式是否為 TIFF、PDF、EPS、PNG 或其他指定格式。
3. 投稿版是否需要 line numbers、double spacing、Letter size 或 A4、2.54 cm margin。
4. 是否需要 Running Title、Highlights、Graphical Abstract、Data Availability Statement、Conflict of Interest 或 CRediT author contribution。
5. Title 是否置中、是否全部大寫、是否粗體；作者姓名順序、姓與名的寫法、affiliation 位置是否符合規定。
6. Abstract 標題是否全部大寫或置中，摘要是否緊接在題名後。
7. 圖的引用格式是 Fig. 1、Fig. 1. 或 Figure 1；句首與句中格式是否不同。
8. 文獻引用格式是 Theis (1935)、Theis [12] 或直接 [12]。
9. 章節是否需要數字編號，小節標題是否斜體。
10. Figure Captions 是放在本文最後、圖下方，或另列 List of Figures。
11. 期刊名是否縮寫、是否斜體，References 第二行是否內縮。
12. 關鍵字須包含能幫助 editor 指派 reviewer 的核心詞。地下水論文可視主題加入 groundwater、aquifer test、pumping test、well hydraulics 等。

### 2.11.3 期刊論文 Word 檔案上傳前後 SOP

#### 上傳前

1. 先檢查期刊 Author Guide，看規定內容是否全部符合。
2. 準備 suggested reviewers。若期刊允許，也可提出 opposed reviewers，但理由必須客觀、專業且可被編輯接受。
3. 所有圖都要準備期刊要求的高解析圖檔。本文若只需保留 Figure Captions 或 List of Figures，就不要把圖錯放進主文檔。
4. 投稿通常包括 cover letter、manuscript Word/PDF 檔、所有圖檔、supplementary materials 與必要聲明。
5. 系統完成上傳後，通常會產生合併檔。必須仔細檢查合併檔內容是否完整、圖表是否出現、Word 檔是否為最後一版、公式與符號是否正常。

#### 上傳後

1. 完成上傳後，保存合併檔、最後 Word 檔、PDF 檔、圖檔與 supplementary files，並放到該論文專案的固定資料夾。
2. 合併檔不只用來確認投稿內容，也可在日後回覆 reviewer 行號問題時對照。
3. 若審查時間異常、系統狀態不明或收到特殊通知，須記錄日期並告知老師或共同作者。
4. 遇到棘手審查意見時，不要急著寫反駁信。先把問題分類，再和老師討論回覆策略與補充分析。

## 2.12 期刊或研討會論文給老師修改稿

1. 給老師修改的版本須依目標期刊或研討會格式設定 paper size。若期刊要求 Letter size 就用 Letter size，否則依規定使用 A4 或系統指定格式。
2. Word 或 PDF 需方便閱讀。若使用追蹤修訂，須確認顯示比例、修訂標記與註解不會遮住文字。
3. 每頁須有頁碼。若多位作者版本流通，可在頁首或頁尾加簡短識別字，避免不同稿件混淆。
4. 須有 List of Figures 或 Figure Captions。給老師的修改稿中，所有圖形下方可暫時加 Figure Caption，方便看稿；若期刊要求圖表分開上傳，外投前再依規定調整。
5. 須提供所有圖檔與相關數據。舊手冊提到 Grapher 與 TIFF；新版可依實際工具保存原始繪圖檔、資料表、程式碼與期刊要求圖檔。
6. 提供 Keywords 時，先列出較完整版本，再依期刊限制刪減。第一個或前幾個關鍵字要能準確反映學科與主題。
7. 交稿時須附上老師需要判斷的問題清單，例如：哪一個模型比較合適、哪張圖要保留、哪個 reviewer 意見需要策略性回應。
8. 給老師修改的文稿不能只給單一乾淨稿；應包含 clean manuscript、marked manuscript、response/change log、圖表來源與主要分析檔案位置。

## 2.13-2.14 寫作參考資料

原手冊後段收錄外部科技寫作建議與 technical writing 材料。新版不逐字公開外部材料，但保留可操作原則：

1. 寫作前先建立 outline，不要等到研究完全結束才開始寫。
2. Introduction 與 Conclusions 要花足夠時間，因為它們決定讀者如何理解研究價值。
3. 好文章可模仿結構、節奏與論證方式，但不可抄句子。
4. 不要把兩個完整好題目硬塞進同一篇文章；一篇文章最好只處理一個清楚問題。
5. 若要轉投，須依新期刊讀者、格式與審稿重點重寫，不可只改標題與格式。
6. 對 reviewer 或前人研究的描述要公平、具體、可被證據支持。
7. 寫作不是包裝，而是把問題、方法、證據、限制與貢獻排列成可檢查的論證。
