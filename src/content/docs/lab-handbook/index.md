# 新生入門

歡迎加入研究室。你不需要一開始就什麼都會，但需要養成可靠的工作習慣：資料要放得回去、圖表要說得清楚、進度要查得到、問題要先整理過再討論。

請先照下面順序完成。讀不懂的地方先標記，不要卡在原地太久。

## 第一週先完成

| 任務 | 你要做到什麼 |
|---|---|
| 讀完基本規則 | 看完 [第 1 章：規則與辦法](./chapter-1-rules/)，先了解會議、檔案、簡報、借用資料與學術倫理。 |
| 建立工作資料夾 | 建立自己的 `data`、`program`、`figure`、`manuscript`、`note` 資料夾，檔名要能看出日期、版本與用途。 |
| 準備每週進度紀錄 | 建立固定的週進度檔，內容至少包含本週做了什麼、遇到什麼問題、下週要做什麼。 |
| 整理文獻清單 | 先放入老師指定或自己找到的文獻，記錄作者、年份、題目、方法、資料與限制。 |
| 確認工具環境 | 確認電腦、帳號、備份、文獻管理、繪圖、程式與文件工具能正常使用。 |

## 第一個月要練會

1. 能把一篇論文講清楚：研究問題、方法、資料、結果、限制，以及和自己題目的關係。
2. 能交出一張可討論的圖：包含資料來源、處理方式、座標軸、單位、圖說與初步解釋。
3. 能把程式與資料整理到別人找得到的位置，不只放在自己的桌面或下載資料夾。
4. 能在討論前先寫下問題，而不是只說「我不會」「我不知道」「跑不出來」。
5. 能做一份短簡報，說明背景、方法、目前結果與下一步。

## 問問題前先準備

請先整理成下面四行，再拿來討論：

```text
我現在想完成：
我已經做了：
我卡住的地方是：
我下一步想試：
```

如果是程式或資料問題，請附上可重現的資料、指令、錯誤訊息與輸出圖。只有口頭描述通常不夠。

## 交資料的基本要求

- 圖表要有檔名、日期、資料來源、單位與圖說。
- 程式要能重新執行，不要只留下暫存結果。
- 文獻筆記要記下作者、年份、題目、方法、主要結論與限制。
- 簡報不要只貼文字，要讓聽的人看得懂你的資料、方法與判斷。
- 用 AI 工具產生的內容要自己查核，不能把沒確認的文字或數據直接交出去。

## 最重要的原則

研究不是一次做對，而是每一步都留下可檢查的痕跡。只要你的資料、程式、圖表、筆記與問題都能被追蹤，研究就能慢慢推進；如果東西散掉、來源不清楚、版本不明，後面會花更多時間補救。
