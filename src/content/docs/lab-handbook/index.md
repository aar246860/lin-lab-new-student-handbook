# 研究室手冊總覽

這份手冊給剛加入研究室的人使用，也給已經開始做題目的學生回頭查規則。它不是一份漂亮但空泛的介紹，而是一套工作方式：你要如何整理資料、留下證據、寫週報、準備 Group Seminar、使用 AI Agent、把抽水試驗與地下水模型推到可以檢查的工程判斷。

手冊的原始參考是一份舊版地下水研究室手冊。新版不直接複製舊內容，而是保留它的詳細度與訓練精神，改寫成林穎凡研究室目前需要的版本。

## 研究室核心精神

研究室不是只追求「把答案算出來」。我們更在意下面幾件事：

1. 資料要能追蹤：原始資料、清理後資料、圖、程式與文字結論要能互相對回去。
2. 模型要講清楚：Theis、Cooper-Jacob、Hantush、Neuman、Lagging Darcy 等解析模型，都不是黑箱按鈕。
3. 參數是判釋結果：抽水試驗量到的是洩降與時間序列；水力傳導係數、貯水係數與 lagging 參數是模型條件下的解釋。
4. 不確定性要推到決策：研究不是停在 RMSE 或最佳參數，而是要說明 pumping limit、recovery time、risk margin 或其他工程端點會不會改變。
5. AI Agent 是加速器，不是證據來源：AI 可以幫你整理、檢查、改寫、寫程式，但不能替你編造資料、文獻或模型結果。

## 你應該先看哪裡

如果你剛加入，建議順序如下：

1. 先看「來源邊界」，知道這份手冊哪些內容是改寫、哪些需要人工確認。
2. 看「日常運作」，了解週報、檔案命名、會議與報告。
3. 看「研究訓練」，了解八週內要練哪些基本能力。
4. 看「論文與發表」，知道研究筆記如何變成 manuscript、研討會摘要或期刊投稿。
5. 看「AI Agent 工作流」，學會怎麼讓 Codex 幫忙，但不失去自己的判斷。
6. 看「Skill Pack」，知道哪些任務未來可以交給可重複調用的技能輔助。

## 加入後第一週要完成

第一週不要求你立刻提出偉大的題目，但要做到幾件具體事情：

- 建立自己的 GitHub 或雲端研究資料夾。
- 交一份 `weekly-note.md`，寫清楚本週做了什麼、卡在哪裡、下週要交什麼。
- 用 Python 或其他可重現方式畫一張簡單圖。
- 用固定格式整理一篇文獻，不要只貼摘要。
- 用 AI Agent 幫你檢查筆記，但最後由你確認內容是否正確。

## 研究路線

本研究室目前的主線可以理解成四層：

| 層次 | 問題 | 對應訓練 |
|---|---|---|
| 水井力學與解析解 | 傳統模型如何從洩降資料推估參數？ | Theis、Cooper-Jacob、Hantush、Neuman、boundary effects |
| Lagging Darcy / Lagging Theory | flux 與 hydraulic gradient 不一定同步時，如何描述非瞬時響應？ | normalized response、lagging 參數、殘差診斷 |
| 轉移不確定性 | 從觀測資料到模型參數再到設計值時，誤差如何放大？ | model-conditioned parameters、model factor、COV、benchmark |
| 決策可靠度 | 參數差異會不會改變工程判斷？ | pumping limit、recovery time、risk margin、scenario audit |

## 使用規則

你可以把這份手冊當成入門地圖，但不要把它當成所有問題的答案。真正的研究仍然要回到資料、文獻、模型推導與可重現計算。

當手冊、老師口頭說明與正式專案規定互相衝突時，以正式專案規定與最新會議紀錄為準。手冊會持續更新，修改時要留下 commit 訊息，說明改了什麼與為什麼改。
