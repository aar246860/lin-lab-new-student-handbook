# 第三章 寫作參考

寫作不是把資料放進文章，而是讓讀者看懂你提出什麼問題、用什麼方法回答、證據支持到哪裡。這一章整理舊手冊中寫作參考書的共同重點，供新生在寫報告、摘要、論文與技術文件時查閱。

## 3.1 先想清楚讀者

寫作前先判斷讀者是誰：

| 讀者 | 需要看到什麼 |
|---|---|
| 老師與研究室同學 | 研究問題、目前進度、方法是否合理、哪裡需要決策。 |
| 研討會聽眾 | 問題背景、方法重點、主要結果、為什麼值得聽。 |
| 期刊審稿人 | 新意、證據、方法透明度、和既有文獻的差異。 |
| 業界或計畫單位 | 決策意義、限制、風險、可執行建議。 |

不要把所有讀者都當成已經懂你題目的人。文章前段要讓讀者知道：這個問題為什麼重要、目前做法有什麼不足、你的工作補上哪一塊。

## 3.2 研究報告的基本結構

一份完整研究報告通常包含：

1. 研究動機與問題。
2. 前人研究與目前缺口。
3. 資料來源與處理方式。
4. 方法、模型或理論。
5. 結果。
6. 討論。
7. 結論與建議。
8. 參考文獻。
9. 附錄或補充材料。

寫作時不要只把章節名稱填滿。每一章都要能回答一個問題。例如：方法章要回答「別人能不能照著重做？」結果章要回答「資料真的顯示了什麼？」討論章要回答「這個結果代表什麼、不能代表什麼？」

## 3.3 科學論文寫作要點

### 題名

題名要讓讀者知道研究對象、方法或主要貢獻。不要使用過度籠統的題名，例如「地下水研究」或「抽水試驗分析」。好的題名通常包含核心問題與研究方法。

### 摘要

摘要應包含問題、方法、資料或案例、主要結果與意義。摘要不是前言縮短版，也不是結論複製版。摘要內盡量避免太多縮寫、文獻引用與方程式。

### 前言

前言需要做到三件事：

1. 說明問題為什麼重要。
2. 說明前人研究已經做到什麼，以及仍有什麼缺口。
3. 清楚寫出本文目的與貢獻。

前言不要只是堆文獻。每一段文獻回顧都要服務於本文問題。

### 方法

方法章要寫到別人可以理解與重做。若使用模型，要交代假設、方程式、參數、邊界條件、資料處理、校正方法與限制。若有程式或資料處理流程，應留下可追溯紀錄。

### 結果與討論

結果章先講資料顯示了什麼，討論章再講這代表什麼。不要在沒有證據的地方過度推論。每一張圖都要能對應到一個句子：這張圖要證明或說明什麼？

### 結論

結論不是摘要重寫。結論應直接回答研究目的，並說明實務意義、限制與下一步。

## 3.4 技術文件寫作

技術文件重點是讓他人能使用、維護與重現。研究室常見技術文件包括程式說明、資料說明、圖表產生流程、模型設定、實驗紀錄與交接文件。

技術文件至少要包含：

1. 目的。
2. 使用情境。
3. 輸入資料。
4. 執行步驟。
5. 輸出結果。
6. 已知限制。
7. 修改紀錄。

不要只寫「跑程式即可」。未來的自己或下一位學生通常不會記得當時的環境設定。

## 3.5 中文作者寫英文科技文

英文科技文不追求華麗，而追求清楚、精準、可檢查。常見問題包括：

| 問題 | 修改方向 |
|---|---|
| 句子太長 | 拆成兩句，讓主詞與動詞清楚。 |
| 主詞模糊 | 寫清楚是 data、model、result、method 或 this study。 |
| 用字太口語 | 改成學術或技術文件常用語。 |
| 時態混亂 | 已完成研究多用過去式；普遍事實多用現在式。 |
| 貢獻說不清楚 | 用一句話說明本文和前人差在哪裡。 |

寫英文時先把中文邏輯想清楚，再處理句型。不要用翻譯軟體把不清楚的中文直接翻成英文。

## 3.6 寫作檢查表

交稿前至少檢查：

- 題名是否準確。
- 摘要是否包含問題、方法、結果與意義。
- 前言是否清楚指出研究缺口。
- 方法是否足以重現。
- 圖表是否有單位、圖說與資料來源。
- 結果是否和圖表一致。
- 討論是否沒有超出證據。
- 結論是否回答研究目的。
- 參考文獻格式是否一致。
- 檔名與版本是否清楚。
