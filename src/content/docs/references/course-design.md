# 課程設計來源

這套教材不是把地下水知識點直接排成章節，而是用幾個教育設計原則組合成暑期訓練路線。

## Backward design

MIT Teaching + Learning Lab 對 backward design 的說明是：先定義學生最後要能做什麼，再決定哪些成果可以作為證據，最後才安排學習活動。這套訓練就是從「最後完成一份決策可靠度報告」往回設計八週 lab。

來源：MIT Teaching + Learning Lab, "Where to Start: Backward Design"
https://tll.mit.edu/teaching-resources/course-design/backward-design/

## Constructive alignment

University of Waterloo Centre for Teaching Excellence 強調學習成果、評量方式和課堂活動需要對齊。因此，每週 lab 都固定列出學習目標、本週任務、繳交成果與通過標準，讓學生知道為什麼要做、要交什麼、怎樣才算達標。

來源：University of Waterloo, "Aligning Outcomes, Assessments, and Instruction"
https://uwaterloo.ca/centre-for-teaching-excellence/catalogs/tip-sheets/aligning-outcomes-assessments-and-instruction

## High-structure course design

NC State DELTA 將 high-structure course design 描述為透過清楚目標、頻繁活動、週期性回饋與學生互動來支持學生成功。新進學生的背景差異很大，所以這套訓練使用高結構設計：步驟要清楚，檢核點要明確，也要允許學生修正後再交。

來源：NC State DELTA, "High Structure Course Design Improves Student Success"
https://news.delta.ncsu.edu/2024/12/19/high-structure-course-design-improves-student-success/

## Transparent assignment design

Ohio State 的 Transparent Assignment Template 使用 purpose、tasks、criteria 來拆解作業。每週 lab 都採用相同格式，降低學生猜題意的成本。

來源：Ohio State Teaching and Learning Resource Center, "Using the Transparent Assignment Template"
https://teaching.resources.osu.edu/examples/using-transparent-assignment

## AI literacy

UNESCO 的 AI competency framework for students 將 AI 能力分成幾個面向，包含以人為本的心態、AI 倫理、AI 技術與應用、AI 系統設計等。這套教材允許學生使用 AI 協助學習，但必須標示資料來源，也要理解模型限制。

來源：UNESCO, "AI competency framework for students"
https://www.unesco.org/en/articles/ai-competency-framework-students

## Website framework

Astro Starlight 是文件網站框架，支援導覽、搜尋、SEO、易讀排版、程式碼高亮與深色模式。這個 repo 使用 Astro Starlight，讓教材可以透過 GitHub Pages 發布。

來源：Astro Starlight
https://starlight.astro.build/

## 正體中文教材語氣

台大「課程大綱優化」建議，課程目標應從學生學習成效出發，使用能評量的動詞，例如「說明」、「操作」、「分析」，避免只寫「認識」或「了解」。因此，每週目標都寫成學生可以完成的動作。

來源：臺灣大學教學發展中心, "課程大綱優化"
https://www.dlc.ntu.edu.tw/optimize/

臺灣易讀資訊的設計精神，是把複雜內容轉成容易理解、容易閱讀的版本。這套教材採用較短的句子、明確的步驟和固定欄位，降低新生進入研究工作的門檻。

來源：臺灣易讀參考指南－讓資訊易讀易懂
https://www.klepb.klcg.gov.tw/tw/klepb1/2795-252968.html

Microsoft Traditional Chinese Style Guide 強調，正體中文內容應該清楚、友善，並使用自然的語法與術語。因此，教材避免把英文標題直接翻成中英混雜的格式。

來源：Microsoft Chinese (Traditional) Style Guide
https://download.microsoft.com/download/2/9/3/293e6d41-ba40-451b-a41e-94bdb6242ae3/zho-twn-StyleGuide.pdf

## 使用邊界

教材中所有地下水案例都分成三類：

- **傳統公式**：例如 Theis 與 Cooper-Jacob，用來理解傳統抽水試驗方法。
- **教學簡化**：為了讓概念更清楚，而簡化的模型或圖解。
- **合成資料**：人工建立的練習資料，不代表現地觀測。

學生在報告中必須標示自己使用的是哪一類資料或模型，不能把教學資料描述成現地成果。
