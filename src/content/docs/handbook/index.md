# 新生手冊

歡迎加入。你不需要一開始就很強，但每週都要留下可以檢查的成果。你可以學得慢，可以問問題，也可以重做；但資料不能亂放，圖不能沒有單位，結論也不能沒有證據。

## 你會學到什麼

暑假結束時，希望你能做到：

1. 整理抽水試驗或水位時間序列資料。
2. 用 Python 畫出乾淨、可解釋的圖。
3. 說明 Theis 與 Cooper-Jacob 方法的基本假設。
4. 比較不同模型得到的參數差異。
5. 判斷一個結果是否足以支撐工程決策。
6. 寫出一份簡短、可重現、沒有誇大語氣的研究報告。

## 基本工具

- GitHub：交作業、留下版本紀錄。
- Python：資料整理、繪圖、基本計算。
- Markdown：寫研究筆記。
- AI 工具：可以協助整理、檢查與改寫，但不能取代你對資料與公式的理解。

## 每週作業格式

每週繳交三樣東西：

1. `notes.md`：你這週做了什麼、卡在哪裡、學到什麼。
2. `figure.png` 或 `figure.svg`：一張能支撐你說法的主圖。
3. `answer.md`：回答本週 lab 的問題，並附上你怎麼得到結論。

## 怎麼問問題

請用這個格式：

```text
我想做的事：
我已經試過：
目前錯誤訊息或卡住的地方：
我猜可能原因：
我希望老師或學長姐幫我確認：
```

## AI 使用規則

可以使用 AI：

- 解釋 Python 錯誤。
- 協助整理筆記。
- 協助檢查圖表標題與單位。
- 協助把口語筆記改成正式段落。

不可以使用 AI：

- 直接產生不存在的資料。
- 編造文獻或引用。
- 把合成資料寫成現地資料。
- 在沒有理解公式的情況下貼上答案。

## 通過精神

這套訓練的原則是「標準要清楚，修正要具體」。如果這週沒有通過，就依照回饋修正後再交。重點不是跟同學比排名，而是讓這週的自己比上週更清楚、更可靠。
